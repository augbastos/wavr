# Graphify Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install Graphify, run it as a pilot on the Wavr repo with an automatic incremental-update hook, evaluate the output against a concrete success criterion, and — only if that criterion is met — repeat the same setup on the Lucky Cat repo in the same pass.

**Architecture:** Graphify is a standalone CLI tool (not code we write) that builds a code-structure graph via local Tree-sitter (free) plus optional LLM semantic extraction (Anthropic, using the existing `ANTHROPIC_API_KEY`). This plan has no application code, no tests in the pytest sense — "verification" means running real commands and inspecting real output against a stated criterion.

**Tech Stack:** `uv` (Python tool installer, already present at `/c/Users/broku/AppData/Roaming/Python/Python314/Scripts/uv`), `graphifyy` (PyPI package, CLI command `graphify`), git.

## Global Constraints

- No `--obsidian` or `--wiki` flags on any `graphify` invocation — output stays in each repo's own `graphify-out/` folder, never written into the Obsidian vault.
- `graphify-out/` must be added to each repo's `.gitignore` before the first commit that could pick it up.
- The git hook (`graphify hook install`) gets installed in both Wavr and Lucky Cat from the start — not deferred to "later if it proves useful."
- LLM backend is Anthropic; `ANTHROPIC_API_KEY` is already set in the environment (confirmed present) — no new key setup.
- Lucky Cat only gets touched if Wavr's pilot meets the success criterion below. If it doesn't, stop after Task 3's evaluation and report back — do not proceed to Lucky Cat and do not guess at a fix.
- Spec reference: `C:\IA\wavr\docs\superpowers\specs\2026-07-03-graphify-setup-design.md` (commit `a90e428`).
